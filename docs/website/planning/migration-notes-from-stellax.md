# Migration Notes From Stellax

ASTRA borrows from Stellax selectively.

## Good things to migrate

- clear documentation as project memory,
- explicit architecture pages,
- validation culture,
- clean state/config naming when it improves clarity,
- formulation-aware thinking.

## Things ASTRA should not inherit directly

- JAX-specific design compromises,
- differentiability-first assumptions in the baseline lane,
- sprawling validation surfaces before the classical Julia core is stable.

ASTRA should learn from Stellax without becoming a line-by-line translation of it.
