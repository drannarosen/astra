# Roadmap

ASTRA's roadmap is about dependency order, not feature excitement.

This page answers a planning question, not a sprint question: what has to become trustworthy before the next layer of ambition is allowed to land?

## Sequence

1. Repository bootstrap
2. Contract-first state and ownership cleanup
3. Classical baseline structure solver
4. Solar-first initialization lane: seed and relaxation
5. Minimal evolution with PMS contraction and ZAMS detection
6. Main-sequence evolution to the solar-age target vector
7. Entropy-DAE experimental lane

The critical planning rule is that classical trustworthiness comes before method competition.

## Solar-first interpretation

The roadmap is now explicitly solar-first:

- first validate the Sun,
- then broaden the physics and mass range,
- then compare alternative formulations.

That is a scope-discipline choice, not a claim that ASTRA only cares about solar-type stars forever.
