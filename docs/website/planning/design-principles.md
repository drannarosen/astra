# Design Principles

ASTRA optimizes for:

1. correctness,
2. clarity,
3. validation,
4. explicit ownership,
5. derivative honesty,
6. extensibility,
7. performance,
8. feature growth.

Those priorities are intentionally not the usual startup-software order. ASTRA is a research code and a teaching artifact, so the architecture has to remain legible long after the first enthusiastic implementation sprint.
