# Checklists

This page is the entry point for ASTRA's developer-facing checklists.

Checklists live in the website because they are part of ASTRA's operational memory, not just private scratch notes. They should be easy for a contributor to find while reading the handbook, reviewing current scope, or checking what has actually been verified.

## How to use checklists

Use a checklist when you need to answer questions like:

- what has been explicitly approved for a lane,
- what has already landed with direct evidence,
- what still remains unimplemented,
- and what is intentionally deferred.

Update a checklist only when there is direct evidence:

- merged code,
- passing tests,
- successful docs builds,
- or an approved design note that changes the accepted roadmap.

Do not check a box because a direction sounds promising in discussion.
