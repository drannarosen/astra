# Dev Journal

The `Dev Journal` is ASTRA's dated development memory lane inside the broader `Development` section.

Use it to answer questions like:

- what changed recently,
- which focused slices produced artifact-backed evidence,
- what the current results do and do not imply,
- and which takeaways should survive after the immediate note becomes historical.

## Pages in this subsection

- [Progress Summary](progress-summary.md): the high-level dated rollup for quick orientation.
- [Lessons Learned](lessons-learned.md): durable takeaways, repeated failure modes, and workflow corrections worth keeping visible after a slice ends.
- Dated topic notes: focused, artifact-backed entries for a specific audit, cutover, or scientific debugging slice.

## Retention workflow

Treat notes in this subsection as one of three kinds:

- `current`: still informs active decisions or current interpretation.
- `historical`: still useful as evidence, but no longer the live guidance surface.
- `archive-candidate`: mostly superseded and ready either for consolidation into `progress-summary.md` or promotion of its durable takeaway into `lessons-learned.md`.

When a note becomes stale, update its retention status explicitly rather than letting it silently accumulate authority.
