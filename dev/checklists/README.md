# Development Checklists

This directory is ASTRA's lightweight execution lane for tracking progress against approved design contracts.

## Purpose

Use these checklists to record:

- what has been approved,
- what has been implemented,
- what has been verified,
- and what remains intentionally out of scope.

The goal is to reduce architectural drift and help ASTRA grow by narrow, validated slices rather than by accumulating half-finished science ideas.

## Current checklists

- `solar-first-lane.md`

## Maintenance rule

Update a checklist only when there is direct evidence:

- merged code,
- passing tests,
- successful docs build,
- or an approved design note that changes the accepted roadmap.

Do not check a box because a direction sounds good in discussion.
